NC News API

This API will retrieve information regarding various users, topics and articles from the backend database based on the various endpoints

  • GET topics
  • GET articles
  • GET users
  • GET users/:username
  • GET articles/:article_id
  • GET articles/:article_id/comments
  • POST articles/:article_id/comments
  • PATCH articles/:article_id
  • PATCH comments/:comment_id
  • DELETE comments/:comment_id

    Sorting Queries

    Retrieve a list of all articles based on the named queries.

    Calling the articles API endpoint without any queries will return the full list of articles. By default the articles will be sorted by their created_at column in descending order. This can be changed by adding a 'sort_by' and 'order' query to the GET request.

    Example: ?sort_by=author will order the articles by their author in reverse alphabetical order (descending order based on UTF-8 character code). ?sort_by=author&order=asc will order the articles by author in alphabetical order.

    Example response snippet:

    Endpoint: /api/articles?sort_by=author&order=asc

  • POST /api/articles/:article_id/comments

    Post a new comment under an article based on the article ID.

    To post a new comment, the body request requires two parameters, username (author) and must be a valid username, and the body (the text within the comment) 

                
                {
                    username: cooljmessy,
                    body: Lorem ipsum dolor sit amet, consectetur adipisicing elit. Quasi possimus ut repellat delectus laudantium pariatur ipsum animi aliquid fuga ratione cum et vitae saepe, necessitatibus facilis explicabo nulla reprehenderit nisi.
                }

    Example response for ID=2:

      
{
	"comment_id": 301,
	"article_id": 2,
	"body": "Lorem ipsum dolor sit amet, consectetur adipisicing elit. Quasi possimus ut repellat delectus laudantium pariatur ipsum animi aliquid fuga ratione cum et vitae saepe, necessitatibus facilis explicabo nulla reprehenderit nisi.",
	"votes": 0,
	"author": "cooljmessy",
	"created_at": "2026-02-10T14:23:49.423Z"
}

    The response from the server returns a formatted comment, with all the required properties.
